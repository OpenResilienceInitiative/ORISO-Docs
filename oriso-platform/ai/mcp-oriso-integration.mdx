## ORISO Platform Relevant MCP Servers

Based on the ORISO platform architecture and tech stack, here are relevant MCP servers that can enhance your development workflow:

### Core Infrastructure

**GitHub** (Official)
- **Purpose**: Repository management, issue tracking, PR management
- **Package**: `@modelcontextprotocol/server-github`
- **Status**: ✅ Already configured (see [Getting Started](/oriso-platform/ai/mcp-getting-started))
- **Where to Find**: 
  - Archived: [servers-archived/src/github](https://github.com/modelcontextprotocol/servers-archived/tree/main/src/github)
  - Install via: `npx -y @modelcontextprotocol/server-github`

**Slack** (Community)
- **Purpose**: Team communication, status updates, notifications
- **Package**: `@modelcontextprotocol/server-slack`
- **Status**: ✅ Already configured (see [Getting Started](/oriso-platform/ai/mcp-getting-started))
- **Where to Find**: 
  - Archived: [servers-archived/src/slack](https://github.com/modelcontextprotocol/servers-archived/tree/main/src/slack)
  - Install via: `npx -y @modelcontextprotocol/server-slack`

**Matrix** (Community)
- **Purpose**: E2EE messaging integration (synergy with ORISO-Matrix)
- **Package**: Community server available

- **Where to Find**: 
  - Community implementations: Check [MCP Servers Registry](https://github.com/modelcontextprotocol/servers) for "matrix"

**Keycloak** (Community)
- **Purpose**: Identity and access management operations
- **Package**: Community MCP servers available

## Recommended Configuration for ORISO

Add these to your `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_your_token"
      }
    },
    "slack": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-slack"],
      "env": {
        "SLACK_BOT_TOKEN": "xoxb-your-token",
        "SLACK_TEAM_ID": "T01234567",
        "SLACK_CHANNEL_IDS": "C01234567"
      }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres", "postgresql://user:pass@localhost/oriso_tenant_db"]
    },
    "kubernetes": {
      "command": "npx",
      "args": ["-y", "@your-org/server-kubernetes"],
      "env": {
        "KUBECONFIG": "~/.kube/config"
      }
    },
    "prometheus": {
      "command": "npx",
      "args": ["-y", "@community/server-prometheus"],
      "env": {
        "PROMETHEUS_URL": "http://localhost:9090"
      }
    }
  }
}
```

<Note>
**Finding MCP Servers:**
- **Official servers**: Check [MCP Servers Registry](https://github.com/modelcontextprotocol/servers) main repository
- **Archived servers**: Check [servers-archived](https://github.com/modelcontextprotocol/servers-archived) repository (PostgreSQL, Redis, GitHub, Slack, etc.)
- **Community servers**: Search the [MCP Servers Registry](https://github.com/modelcontextprotocol/servers) README for community implementations
- **Installation**: Most servers can be installed via `npx -y @package-name` or `npm install`, but installation methods may vary

**Note on Archived Servers**: Some servers like PostgreSQL and Redis have been moved to the archived repository. You can still use them, but consider checking for newer community implementations in the main registry.
</Note>

## Example Workflows

### Morning Standup Automation

**Daily Automation Script:**
```
Every morning at 9 AM:
1. Fetch all GitHub issues updated in last 24 hours
2. Fetch all pull requests opened/merged in last 24 hours
3. Summarize code review activity
4. Format as standup update
5. Post to #daily-standup Slack channel
```

### Release Checklist

**Release Workflow:**
```
1. List all pull requests merged to main branch
2. Generate release notes summary
3. Check for breaking changes
4. Post release summary to #releases Slack channel
5. Create GitHub release with summary
```

### Review Queue Management

**Code Review Workflow:**
```
1. Find PRs ready for review (CI passed, no blocking reviews)
2. Check reviewer assignments
3. Notify team in #code-review Slack channel
4. Update daily review queue dashboard
```
