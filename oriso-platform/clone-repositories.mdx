# Clone ORISO Repositories

Set up the project directory and transfer ORISO repositories to your server.

<Info>
**New Option**: ORISO-Debian now contains all systems in one repository. If you want to clone the whole system at once, you can reference and use the [ORISO-Debian repository](https://github.com/your-org/ORISO-Debian) which includes all ORISO components in a single repository.
</Info>

<Steps>
<Step title="Create Project Directory">

Create the main project directory for ORISO Platform files.

<Tabs>
  <Tab title="Ubuntu/Debian">

```bash
mkdir -p ~/online-beratung/caritas-workspace
cd ~/online-beratung/caritas-workspace
git init
```

  </Tab>
  <Tab title="macOS">

```bash
mkdir -p ~/online-beratung/caritas-workspace
cd ~/online-beratung/caritas-workspace
git init
```

  </Tab>
  <Tab title="Windows">

```powershell
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\online-beratung\caritas-workspace"
Set-Location "$env:USERPROFILE\online-beratung\caritas-workspace"
git init
```

  </Tab>
</Tabs>

</Step>

<Step title="Transfer ORISO Repositories">

Choose one of the following methods to transfer your ORISO repositories to the new server.

<Tabs>
<Tab title="Method 1: Clone ORISO-Debian (Recommended) ⭐ NEW">

**ORISO-Debian contains all systems in one repository** - the easiest way to get the complete system.

```bash
# Clone ORISO-Debian (contains all ORISO components)
cd ~/online-beratung
git clone <your-oriso-debian-repo-url> ORISO-Debian

# Navigate to caritas-workspace (all components are here)
cd ORISO-Debian/caritas-workspace

# Verify all components are present
ls -la

# Should see all ORISO-* directories:
# ORISO-Kubernetes/
# ORISO-Database/
# ORISO-Keycloak/
# ORISO-Frontend/
# ORISO-Admin/
# ORISO-TenantService/
# ... and all other components
```

<Info>
**ORISO-Debian** is a complete repository containing all ORISO Platform components in one place. This is the recommended method for cloning the entire system.
</Info>

</Tab>

<Tab title="Method 2: SCP (From Existing Server)">

Transfer directly from an existing server using SCP.

```bash
# On NEW server, from OLD server
scp -r old-server:/path/to/caritas-workspace ~/online-beratung/

# Verify
ls -la ~/online-beratung/caritas-workspace/
```

</Tab>

<Tab title="Method 3: Git Repository (Individual Repos)">

Clone individual repositories if you prefer separate repos.

```bash
# Clone from Git (individual repositories)
cd ~/online-beratung
git clone <your-oriso-repo-url> caritas-workspace

# Verify
ls -la ~/online-beratung/caritas-workspace/
```

</Tab>

<Tab title="Method 4: Manual Transfer (tar.gz)">

Create an archive on the old server and transfer it.

**On OLD server:**

```bash
cd /path/to/online-beratung
tar -czf oriso-complete.tar.gz caritas-workspace/
```

**Transfer to NEW server:**

```bash
scp oriso-complete.tar.gz new-server:~/
```

**On NEW server:**

```bash
cd ~/online-beratung
tar -xzf ~/oriso-complete.tar.gz

# Verify
ls -la ~/online-beratung/caritas-workspace/
```

</Tab>
</Tabs>

</Step>

<Step title="Verify Repository Structure">

Verify all required repositories and directories are present.

<Tabs>
  <Tab title="If using ORISO-Debian">

```bash
# If you cloned ORISO-Debian
cd ~/online-beratung/ORISO-Debian/caritas-workspace
ls -la
tree -L 2 ORISO-Kubernetes/ || find ORISO-Kubernetes -maxdepth 2 -type d
```

</Tab>
  <Tab title="If using individual repos">

```bash
# If you cloned individual repositories
cd ~/online-beratung/caritas-workspace
ls -la
tree -L 2 ORISO-Kubernetes/ || find ORISO-Kubernetes -maxdepth 2 -type d
```

</Tab>
  <Tab title="Windows">

```powershell
# For ORISO-Debian
Set-Location "$env:USERPROFILE\online-beratung\ORISO-Debian\caritas-workspace"
# OR for individual repos
# Set-Location "$env:USERPROFILE\online-beratung\caritas-workspace"
Get-ChildItem
Get-ChildItem -Path ORISO-Kubernetes -Recurse -Depth 2 | Select-Object FullName
```

</Tab>
</Tabs>

<Check>
You should see the following directories:
- `ORISO-Kubernetes/`
- `ORISO-Database/`
- `ORISO-Keycloak/`
- `ORISO-Redis/`
- `ORISO-Nginx/`
- `ORISO-Matrix/`
- `ORISO-SignOZ/` (optional)
- `ORISO-Element/`
- `ORISO-HealthDashboard/`
- `ORISO-Frontend/`
- `ORISO-Admin/`
- `ORISO-TenantService/`
- `ORISO-UserService/`
- `ORISO-AgencyService/`
- `ORISO-ConsultingTypeService/`
</Check>

<Check>
Expected structure:
```
ORISO-Kubernetes/
├── README.md
├── DEPLOYMENT.md
├── STATUS.md
├── deployments/
├── services/
├── configmaps/
└── scripts/
```
</Check>

<Info>
**Note**: If you used ORISO-Debian, all components are in `ORISO-Debian/caritas-workspace/`. If you used individual repos, they are directly in `caritas-workspace/`.
</Info>

</Step>
</Steps>
