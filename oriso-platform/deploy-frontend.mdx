# Deploy Frontend

Deploy the frontend and admin web applications.

<Steps>
<Step title="Deploy Frontend and Admin">

Deploy both frontend and admin applications.

```bash
cd ~/online-beratung/caritas-workspace/ORISO-Kubernetes

# Deploy frontend
kubectl apply -f deployments/05-frontend.yaml

# Wait for frontend
kubectl wait --for=condition=ready pod -l tier=frontend -n caritas --timeout=300s

# Check status
kubectl get pods -n caritas | grep -E "frontend|admin"
```

<Check>
You should see both pods running:
- frontend-xxx    1/1     Running
- admin-xxx       1/1     Running
</Check>

</Step>

<Step title="Verify Frontend Access">

Verify frontend and admin are accessible.

```bash
# Check Frontend (should return HTML)
curl -I http://127.0.0.1:9001
# Expected: HTTP/1.1 200 OK

# Check Admin (should return HTML)
curl -I http://127.0.0.1:9002
# Expected: HTTP/1.1 200 OK

# Access in browser
SERVER_IP=$(hostname -I | awk '{print $1}')
echo "Frontend: http://$SERVER_IP:9001"
echo "Admin: http://$SERVER_IP:9002"
```

<Check>
Both endpoints should return HTTP 200 OK status.
</Check>

</Step>

<Step title="Configure frontend authentication settings">

Point the apps to Keycloak and verify the login flow.

1) Identify your Keycloak public URL and realm

```bash
SERVER_IP=$(hostname -I | awk '{print $1}')
echo "Auth base (via Nginx): http://$SERVER_IP:8089/auth"
echo "Realm: online-beratung"
```

2) Ensure the frontend uses the correct values (client id defaults to `app` unless changed)

- Auth URL: `http://YOUR_SERVER_IP:8089/auth`
- Realm: `online-beratung`
- Client ID: `app`

3) Smoke test login

- Open Frontend `http://YOUR_SERVER_IP:9001`
- Click Login → You should be redirected to Keycloak at `http://YOUR_SERVER_IP:8089/auth/...`
- Login with a test user (or admin) and return to the app

<Warning>
If you see `HTTPS required` or redirect/CORS errors, re‑run Keycloak HTTP configuration and set valid redirect URIs/Web Origins in the realm client. See Configure Keycloak.
</Warning>

</Step>
</Steps>
