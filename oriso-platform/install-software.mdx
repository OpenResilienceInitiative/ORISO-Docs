# Install Required Software

Install all required software tools needed for ORISO Platform deployment, including Docker, Kubernetes, and database clients.

<Steps>
<Step title="Install Docker">

Install Docker for building container images if needed.

<Tabs>
  <Tab title="Ubuntu/Debian">

```bash
# Remove old versions
sudo apt remove -y docker docker-engine docker.io containerd runc

# Add Docker GPG key
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# Add Docker repository
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Install Docker
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Add user to docker group
sudo usermod -aG docker $USER

# Apply group change (or logout/login)
newgrp docker

# Verify Docker installation
docker --version
docker compose version

# Test Docker
docker run hello-world
```

  </Tab>
  <Tab title="macOS">

```bash
# Install Docker Desktop
brew install --cask docker || true
# Start Docker Desktop manually

docker --version
docker compose version
```

  </Tab>
  <Tab title="Windows">

```powershell
winget install --silent Docker.DockerDesktop
# Start Docker Desktop manually

docker --version
docker compose version
```

  </Tab>
</Tabs>

<Check>
Verify Docker is working: `docker --version` and `docker compose version` should show installed versions. The hello-world test should complete successfully.
</Check>

<Note>
You may need to log out and log back in for docker group membership to take effect, or use `newgrp docker`.
</Note>

</Step>

<Step title="Install Kubernetes (k3s or Desktop)">

Install k3s on Ubuntu, or use a desktop Kubernetes on macOS/Windows.

<Tabs>
  <Tab title="Ubuntu/Debian — k3s">

```bash
curl -sfL https://get.k3s.io | sh -
sudo systemctl status k3s
sudo chmod 644 /etc/rancher/k3s/k3s.yaml
mkdir -p ~/.kube
sudo cp /etc/rancher/k3s/k3s.yaml ~/.kube/config
sudo chown $USER:$USER ~/.kube/config
sudo k3s kubectl get nodes
```

  </Tab>
  <Tab title="macOS — Rancher Desktop">

```bash
brew install --cask rancher
# Launch Rancher Desktop and ensure Kubernetes is enabled
kubectl config get-contexts
```

  </Tab>
  <Tab title="Windows — Rancher Desktop">

```powershell
winget install --silent Rancher.RancherDesktop
# Launch Rancher Desktop and enable Kubernetes
kubectl config get-contexts
```

  </Tab>
</Tabs>

<Check>
Expected output should show your node with status `Ready` and roles `control-plane,master`:
```
NAME             STATUS   ROLES                  AGE   VERSION
oriso-platform   Ready    control-plane,master   1m    v1.x.x
```
</Check>

</Step>

<Step title="Install kubectl">

Install kubectl, the Kubernetes command-line tool.

<Tabs>
  <Tab title="Ubuntu/Debian">

```bash
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl
rm kubectl
kubectl version --client
kubectl get nodes
```

  </Tab>
  <Tab title="macOS">

```bash
brew install kubectl
kubectl version --client
kubectl get nodes
```

  </Tab>
  <Tab title="Windows">

```powershell
winget install --silent Kubernetes.kubectl
kubectl version --client
kubectl get nodes
```

  </Tab>
</Tabs>

<Check>
Verify kubectl can connect to k3s: `kubectl get nodes` should show your node.
</Check>

</Step>

<Step title="Install Helm (Required)">

Install Helm for managing Kubernetes applications. **Helm is now required** for MariaDB deployment with automatic schema initialization.

<Warning>
Helm is required for MariaDB Helm chart deployment. The ORISO MariaDB Helm chart is included in `ORISO-Kubernetes/helm/charts/mariadb/` and doesn't require external Helm repositories.
</Warning>

<Tabs>
  <Tab title="Ubuntu/Debian">

```bash
# Install Helm
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

# Verify Helm
helm version

# Add Helm repos (for optional components like SignOZ)
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo update
```

<Note>
**Note**: ORISO MariaDB Helm chart is included in `ORISO-Kubernetes/helm/charts/mariadb/` and doesn't require external Helm repositories.
</Note>

  </Tab>
  <Tab title="macOS">

```bash
brew install helm
helm version
```

  </Tab>
  <Tab title="Windows">

```powershell
winget install --silent Helm.Helm
helm version
```

  </Tab>
</Tabs>

<Check>
Verify Helm installation: `helm version` should show the installed version.
</Check>

</Step>

</Step>
</Steps>

## Verification

After completing all installation steps, verify all tools are working:

```bash
# Check Docker
docker --version && docker compose version

# Check Kubernetes
kubectl version --client
kubectl get nodes

# Check k3s service
sudo systemctl status k3s

# Check database clients
mysql --version
mongosh --version
redis-cli --version
```

## Troubleshooting

### k3s Not Starting

Check k3s logs:

```bash
sudo journalctl -u k3s -f
```

Verify swap is disabled (required for k3s):

```bash
free -h
```

### kubectl Can't Connect

Verify kubeconfig:

```bash
cat ~/.kube/config
```

Check k3s API server:

```bash
sudo systemctl status k3s
```

### Docker Permission Denied

Ensure you're in the docker group:

```bash
groups | grep docker
```

If not, log out and log back in, or use:

```bash
newgrp docker
```
