## 10.5 Things the Codebase Confirms

These are *not* assumptions — they're directly verified.

- **14 Keycloak realm roles** exist (`realm.json` + `UserRole.java`).
- **4 backend services** (TenantService 8081, UserService 8082, ConsultingTypeService 8083, AgencyService 8084), Spring Boot 2.7.14 + Java 17.
- **Matrix Synapse** is the chat backbone; **PostgreSQL** is its DB.
- **MariaDB** holds the relational truth; **MongoDB** holds rich topic config.
- **Helm umbrella chart** named `oriso-platform` deploys all 21 services.
- **Liquibase is disabled** — schemas are managed centrally in `ORISO-Database`.
- **Authority.java** maps roles to granted authorities (line-by-line authority spec in [Section 6.4.2](/product/internal-functionality#6-4-2-authz-spring-security)).

## 10.7 What Should Be the Single Source of Truth Going Forward

In order of authority:

1. **Production code** in the relevant repository (`ORISO-UserService`, `ORISO-AgencyService`, etc.).
2. **`ORISO-Docs/oriso-platform/*`** for infra, deployment and operational details.
3. **This `product/*` set** for product / UX / role / data-model reasoning.
4. **Figma files** for screen-by-screen visual truth.
5. The **huddle transcripts** as the historical record of design decisions.

If two of the above disagree, **fix the lower-numbered source** and let this docs set follow.
